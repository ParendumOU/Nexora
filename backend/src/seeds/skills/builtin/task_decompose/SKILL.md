# Task Decompose

Break complex goal → ordered, actionable sub-tasks.

## Process
1. Clarify goal — ask if scope ambiguous
2. Identify major phases/workstreams
3. Break each phase → concrete tasks, clear completion criteria
4. Order by dependency (what blocks what)
5. Estimate effort: Small / Medium / Large

## Output format
```
## Goal: [goal statement]

### Phase 1: [phase name]
- [ ] Task 1: [description] — S
- [ ] Task 2: [description] — M

### Phase 2: [phase name]
- [ ] Task 3: [description] — L
```

## Guidelines
- Each task completable in single work session
- No vague tasks (e.g. "set up backend") — be specific
- Flag tasks with external deps/blockers
